Analizador de ofertas de empleo
texto → JSON estructurado

Un mini-servicio de terminal que recibe el texto de una oferta y devuelve sus datos en JSON fiable, llamando al LLM por SDK directo — sin frameworks de por medio. Primer artefacto en producción de la ruta Data Science → AI Engineering.

Qué hace
De texto caótico a datos manejables

Un LLM por defecto devuelve texto libre, distinto en cada llamada e inservible para un programa. Este artefacto fuerza una salida con forma fija (un molde Pydantic) y la valida antes de devolverla.

El patrón
Schema → parse → objeto validado

El molde define qué campos quiero; el SDK obliga al modelo a rellenarlo y lo valida. Lo único específico del dominio son los campos y sus descripciones — el motor es reutilizable para facturas, tickets o contratos.

analizar_oferta.pyPydantic
class Oferta(BaseModel):
    puesto: str = Field(description="Título del puesto")
    stack: list[str] = Field(description="Tecnologías mencionadas")
    salario_min: Optional[int] = Field(description="Salario mínimo EUR, o null")
    # ...

respuesta = client.messages.parse(
    model="claude-opus-4-8",
    messages=[{"role": "user", "content": texto}],
    output_format=Oferta,        # ← se pasa el molde
)
oferta = respuesta.parsed_output  # ← objeto validado, no texto
Dos proveedores, mismo patrón
Claude y OpenAI, lado a lado

La misma extracción con los dos SDKs. Cambian tres nombres de parámetro y el id del modelo; el concepto es idéntico. El SDK por debajo es casi igual — todo lo demás es conveniencia.

Por qué este artefacto primero

Antes de tocar frameworks, monté la llamada cruda. Si solo dominas la capa de LangChain, dominas algo que puede desaparecer; si entiendes el SDK por debajo, todo lo demás es conveniencia.
