Agente de búsqueda de empleo

Un agente con tool use real: lee una oferta, la contrasta con tu CV y redacta el encaje.

01 Qué es (y por qué no es otro RAG)

El Artefacto 2 (RAG DOBE) es de un solo paso: pregunta → recupera → responde → fin. Este es un agente: tiene varias herramientas y un bucle que decide cuál usar, en qué orden, y cuándo ha terminado. El camino no está fijado: lo traza el modelo.

Tú: [pegas una oferta] │ ┌───▼──────────────────────────────────────────┐ │ EL AGENTE (bucle piensa → actúa → observa) │ │ │ │ analizar_oferta(texto) → requisitos │ │ match_cv(requisitos) → encaje vs tu CV │ │ redactar_borrador(...) → carta / resumen │ └───┬───────────────────────────────────────────┘ │ Respuesta: encaje + qué te falta + borrador

02 Las tres herramientas

HerramientaQué haceDevuelve
analizar_oferta(texto)Extrae los requisitos reales de una oferta (skills, seniority, imprescindibles vs deseables)lista estructurada de requisitos
match_cv(requisitos)Contrasta esos requisitos con tu CV: qué cumples, qué te falta, dónde encajas fuerteencaje + huecos
redactar_borrador(...)Escribe un borrador de encaje o carta a partir del análisistexto listo para revisar

03 Dos decisiones de diseño ya tomadas

El CV es el conocimiento del agente, no una herramienta. La fuente de verdad es CV_denys_porynets_v4.html; se lee para poder comparar. Vive aislado en datos_personales/ (blindado en .gitignore: lleva datos de contacto).

La herramienta redactar_borrador lleva tus reglas horneadas: narrativa cualitativa por encima de métricas · solo el GPA 8.57 permitido · nada de saturar con números. El agente encodea tus preferencias reales — eso, en una entrevista, demuestra criterio, no solo saber llamar una API.

04 El motor: SDK de OpenAI a mano

Construimos el agent loop nosotros con function calling puro, no con LangGraph. Motivo: ver el bucle en vez de dejarlo en una caja opaca, y reutilizar el mismo stack OpenAI del Artefacto 2 sin dependencias nuevas. Historia de CV: «sé qué hace LangGraph por debajo porque lo he escrito yo».

while True:
    resp = llm.chat(mensajes, tools=TOOLS)   # el modelo responde
    if not resp.tool_calls:                  # ¿ya no pide herramientas?
        break                                #   → END
    for tc in resp.tool_calls:               # por cada herramienta pedida:
        resultado = ejecutar(tc)             #   la corro yo (Python)
        mensajes.append(resultado)           #   se lo devuelvo al modelo
# tú controlas cada vuelta
