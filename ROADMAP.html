RAG desplegado sobre el Corpus DOBE

0/0 ▸
Tu pieza diferenciadora: las 417 páginas del transcript TEA1 convertidas en conocimiento consultable. Nadie más tiene este dataset. Conecta DS-NEXUS con AI Engineering de forma literal.
Artefacto

MES 2–3
Harness de evaluación
Dataset de eval · LLM-as-judge · detección de regresiones
0/0 ▸
Esta es la pieza que más te distingue en el mercado, y por eso va aquí y no al final. Construir un RAG es fácil; saber si funciona y detectar cuándo una nueva versión empeora es lo que separa portfolio de prototipo.
Artefacto
Un script que puntúa tu RAG contra un dataset de eval y te avisa cuando una versión nueva empeora respecto a la anterior.
Definición de "hecho"Cambias algo del RAG, corres el harness y obtienes un número que dice si mejoró o empeoró. La evaluación a ojo no cuenta.
Dataset de eval: 20–30 pares pregunta / respuesta esperada del corpus
LLM-as-judge que puntúa fidelidad (¿la respuesta sale de la fuente?) y relevancia
Reporte que compara versiones y marca regresiones

Agente con tool use real
Function calling · orquestación · manejo de errores
0/0 ▸
Ya tienes intuición aquí por Rex+Chef en DS-NEXUS. Esta fase le pone tool use formal: un agente que ejecuta herramientas reales, no que solo conversa. El RAG del Artefacto 2 se convierte en una de sus herramientas.
Artefacto
Rex+Chef formalizado como agente con function calling: ejecuta 2–3 herramientas reales (una de ellas, consultar el RAG DOBE).
Definición de "hecho"Le das una tarea y el agente decide qué herramienta llamar, la ejecuta y usa el resultado. Si las herramientas son simuladas, no está hecho.
Function calling / tool use sobre Rex+Chef con SDK (Claude tool use y OpenAI)
Orquestación de 2–3 herramientas reales (incluida la búsqueda en el RAG DOBE)
Manejo de errores y reintentos cuando una herramienta falla

MES 4–5
Endurecimiento: prompts + structured outputs
Few-shot · CoT · versionado · medido con el harness
0/0 ▸
Técnicas que se aplican el mismo día que las aprendes — pero aquí con disciplina: cada cambio de prompt se mide con el harness del Artefacto 3. Prompt engineering sin evaluación es superstición.
Artefacto
El agente versiona sus prompts y devuelve siempre estructura válida; cada cambio queda justificado por su impacto medido en la métrica.
Definición de "hecho"Puedes señalar un cambio de prompt y enseñar el número de eval que lo justifica. "Parece mejor" no cuenta.
Few-shot y chain-of-thought aplicados donde mueven la métrica
Structured outputs con validación de schema en todo el pipeline
Versionado de prompts + comparación A/B con el harness de eval
