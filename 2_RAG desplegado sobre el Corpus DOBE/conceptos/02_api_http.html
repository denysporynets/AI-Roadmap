El motor, ahora por la red

API HTTP · FastAPI + uvicorn + token · 29 jun 2026 · protocolo ds-nexus
En una frase: envolver el motor RAG en un servicio web para que cualquier programa pueda preguntarle por la red (no solo tú desde la terminal), con una puerta cerrada por token y el motor cargado una sola vez al arrancar.

Qué cambió (y qué NO)

En la Fase 1 separaste motor (la lógica, rag_core.py) de interfaz (cómo se pide y se presenta). La Fase 2 le añade al motor su primera interfaz de red: api.py. Lo importante para tu CV: no tocamos ni una línea de rag_core.py. La API solo lo envuelve. Esa es la recompensa de haber separado motor de interfaz — cada interfaz nueva es trabajo que se suma, no una reescritura.

Por qué el lifespan es la pieza clave

Cargar el motor (índice + corpus + cliente OpenAI) cuesta segundos. Si lo hicieras en cada pregunta, cada petición arrastraría esa carga → lentísimo. El lifespan lo hace una sola vez, al levantar el servidor, y lo guarda en app.state. Después, cada pregunta solo usa el motor ya caliente. Es exactamente el principio "motor vs interfaz" de la Fase 1, ahora aplicado al arranque del servicio.
El restaurante. No enciendes el horno y picas la cebolla cuando entra cada cliente: lo haces una vez al abrir (eso es el lifespan). Luego cada comanda se sirve rápido porque la cocina ya está caliente.

El token, sin secretos en el código

El token se busca primero en el entorno (os.environ["APP_TOKEN"], que en la nube lo pondrá Secret Manager) y solo si no está, se cae al .env local. Mismo patrón que la clave de OpenAI. Así el mismo código corre en tu Mac y en Cloud Run sin tocar nada, y la clave secreta nunca viaja dentro del código fuente.

Por qué /salud no pide token. El portero del hotel deja que cualquiera asome la cabeza para ver si está abierto; pero para entrar a las habitaciones (/preguntar) necesitas la llave. Cloud Run necesita "asomar la cabeza" cada poco para saber si tu contenedor sigue sano — por eso ese sondeo va sin llave.

Anatomía mínima (lo esencial de api.py)

# el lifespan carga el motor UNA vez al arrancar
@asynccontextmanager
async def lifespan(app):
    app.state.motor = MotorRAG()          # caro: solo aquí
    app.state.token = _cargar_token()     # entorno → .env
    yield

# guardián: si el X-Token no cuadra, corta con 401
def exigir_token(x_token = Header(default=None)):
    if x_token != app.state.token:
        raise HTTPException(401, "Token inválido o ausente.")

@app.post("/preguntar")
def preguntar(p: Pregunta, _=Depends(exigir_token)):
    return app.state.motor.preguntar(p.consulta)   # MISMO dict del motor

Para el CV (cómo contarlo)

Expuse un motor RAG como microservicio HTTP con FastAPI + uvicorn, con carga del modelo una sola vez vía lifespan (no por petición), health check sin auth para orquestadores y endpoint de inferencia protegido por token leído del entorno (compatible con Secret Manager), sin secretos en el código.
