RAG DOBE en producción real

FastAPI + Docker + GCP Cloud Run · protocolo DS-NEXUS · Artefacto 2 · 28 jun 2026

Contexto

El motor RAG ya está completo y verificado por CLI (22_buscar.py retrieval+umbral, 23_responder.py síntesis con cita de fuentes). El objetivo: convertirlo en algo en producción real, con URL pública, para enseñar al mundo como pieza de portfolio — cerrando la "definición de hecho" del Artefacto 2 y tu fase LLMOps.

Decisiones tomadas:

  • Acceso: demo REAL (tu corpus) pero protegido por token que compartes selectivamente. URL viva, no abierta a cualquiera.
  • Despliegue: GCP Cloud Run (proyecto mlops-nuclio-denys), imagen construida en la nube con Cloud Build — sin Docker local.

⚠️ Riesgo crítico que motiva la Fase 0

El corpus es tu proceso de autodescubrimiento TEA (lo más personal que tienes) y está en la carpeta del repo público AI-Roadmap, sin trackear pero TAMPOCO ignorado. Un git add . despistado lo expondría. Hay que blindarlo antes de nada.

🏎️ Principio rector

"Ferrari para carrera de karts, NO." Separar motor (lógica RAG) de interfaz (CLI / API) es buena arquitectura, no sobre-ingeniería. Cada fase termina con verificación; podemos parar entre fases (ritmo pausado).

Qué necesitamos (prerequisitos)

  • gcloud autenticado en mlops-nuclio-denys con billing activo y APIs habilitadas (Cloud Run, Cloud Build, Secret Manager, Artifact Registry).
  • Elegir región (sugerencia: europe-southwest1 = Madrid).
  • Elegir el token de acceso (cadena secreta → Secret Manager, nunca en el código).
  • La imagen Docker llevará el corpus dentro → el registro (Artifact Registry) queda privado y /preguntar exige token.

Fase 1 — Motor reutilizable rag_core.py + CLI fina

  • Extraer a rag_core.py la lógica de 23_responder.py: carga del índice + corpus, recuperar(), bandas de confianza (mismo caveat n=8), y preguntar().
  • preguntar(consulta) → dict estructurado (no print): {consulta, banda, sim_top, respuesta, fuentes[], cercanos}.
  • Clave OpenAI: os.environ primero (Cloud Run), fallback a .env (local).
  • 23_responder.py queda como CLI fina sobre el motor (misma UX 🟢🟡⚪ + 📎 Fuentes).
Verificación: el CLI da idéntico a hoy en las 3 ramas (claro / vecindario / ⚪).

Fase 2 — API local api.py (FastAPI + token)

Nombre SIN prefijo numérico: uvicorn necesita un módulo importable api:app.

  • lifespan: instancia el motor una vez al arrancar.
  • GET /salud → health check sin token (para Cloud Run).
  • POST /preguntar → JSON del motor; protegido por cabecera X-Token vs APP_TOKEN; 401 si falla.
  • GET / → sirve el frontend estático.
Verificación local: uvicorn api:app; /salud OK; token bueno → respuesta; token malo → 401; "ropa boda" → ⚪.

Fase 3 — Frontend static/index.html (tu línea visual)

  • Chat minimalista: campo de token (en localStorage), caja de pregunta, render de respuesta + banda de confianza (🟢🟡⚪) + fuentes (#num · pág).
  • Estética portfolio (dark #0d1117, ámbar #f0a500, Fraunces/DM Sans/DM Mono), montado por FastAPI con StaticFiles.
Verificación local: abrir localhost:8080, meter token, preguntar, ver respuesta + fuentes.

Fase 4 — Empaquetado Docker

  • requirements.txt: fastapi, uvicorn[standard], openai, numpy (pin).
  • Dockerfile: python:3.13-slim → copia motor + api + static + corpus_dobe_enriquecido.json + embeddings/indice_a.* → pip install → uvicorn api:app --port $PORT.
  • .dockerignore: excluye .env, .venv, construccion_corpus/, FASE2/, HTML/MD de resultados, vectores_*.npy pesados. La imagen lleva SOLO lo mínimo.
Verificación: revisar contexto de build (no se cuela .env); el build real se valida en Fase 5 vía Cloud Build.

Fase 5 — Despliegue en Cloud Run

  • Habilitar APIs (run, cloudbuild, secretmanager, artifactregistry) si faltan.
  • Crear secretos OPENAI_API_KEY y APP_TOKEN en Secret Manager.
  • gcloud run deploy dobe-rag --source . --region europe-southwest1 --allow-unauthenticated --set-secrets … → red pública para ENTRAR a la web; /preguntar sigue cerrado por token a nivel de app (= "demo real protegido por token").
Verificación end-to-end: URL pública viva; /salud OK; con token → respuesta+fuentes; sin token → 401; "ropa boda" → ⚪; escala a cero; registro privado.
