Harness de evaluación
el RAG deja de calificarse a ojo

Un banco de pruebas que puntúa el RAG contra un dataset de casos "verdad de campo", usa jueces LLM para medir fidelidad y relevancia, compara cada versión con la anterior y bloquea las regresiones con una puerta de calidad. Cuando tocas algo del RAG, obtienes un número que dice si mejoró o empeoró — no una opinión. Tercer artefacto de la ruta Data Science → AI Engineering.

Qué hace
Medir el RAG, no opinar sobre él

Construir un RAG es fácil; saber si funciona y detectar cuándo una versión nueva empeora es lo que separa un portfolio de un prototipo. El harness convierte esa pregunta en un pipeline con dos velocidades de coste: se evalúa una vez (caro) y se testea mil veces (gratis).

La decisión de diseño
Una sola definición de "regresión"

El informe que se lee y la puerta que bloquea comparten la misma lógica: el test de calidad importa y reutiliza el comparador, no reimplementa su propio criterio. Así es imposible que el informe diga "todo bien" mientras el test dice "roto". Una definición, dos caras: un informe legible y un exit code que un pipeline entiende.

test_calidad.py · el corazón de la puertareutiliza comparar.py
import comparar   # la MISMA lógica que produce el informe

def test_ninguna_alucinacion_nueva(diff):
    # responder algo AUSENTE del corpus = alucinación.
    # es la métrica que distingue a este RAG (sabe callarse),
    # por eso tiene su propio assert dedicado.
    alucinaciones = [r for r in diff["regresiones"] if "FALSO POSITIVO" in r]
    assert not alucinaciones
Por qué un harness gana su sueldo
Cazó una alucinación real y la arregló

No es una demo de juguete: al correr el harness sobre el RAG en producción, detectó un falso positivo. La pregunta "irse de casa" —ausente del corpus— se colaba como respondida porque el umbral de confianza estaba blando (0.40). El prompt no llegó a alucinar, pero la puerta avisó de que el umbral dejaba pasar lo que no debía.

Cómo está construido
Cinco piezas, una red de seguridad
dataset
dataset_eval.json — casos "golden" etiquetados a mano: preguntas directas, de síntesis, trampas (ausentes) y de frontera. Arranca pequeño y crece.
jueces
eval_jueces.py — dos jueces LLM (gpt-4o, superior al generador para evitar auto-preferencia) puntúan faithfulness y relevancia, temperatura 0, salida JSON forzada.
evaluación
evaluar.py — corre el RAG sobre el dataset, mide decisión-umbral + fidelidad + relevancia + citas y emite un informe JSON+HTML. Es el paso que cuesta API.
comparación
comparar.py — fija una línea base y detecta regresiones contra ella con tolerancias. Prioriza la decisión rota sobre las medias: caza la alucinación aunque los promedios suban.
puerta
test_calidad.py — envuelve el comparador en asserts de pytest. Verde si nada empeoró, rojo si algo cayó. No gasta API: solo lee dos JSON.
Lo que se entregó
Los criterios, cumplidos
Dataset de evaluación etiquetado a mano
Casos golden con respuesta esperada: directas, síntesis, trampas ausentes y frontera. Empieza pequeño, ampliable.
LLM-as-judge para fidelidad y relevancia
Juez más fuerte que el generador para evitar auto-preferencia; temperatura 0 y JSON forzado para que la nota sea estable.
Reporte que compara versiones y marca regresiones
Línea base + detección de regresiones con tolerancias; prioriza la decisión rota por encima de las medias.
Puerta de calidad con pytest (extra)
Quality gate que reutiliza el comparador y se pone roja ante una alucinación nueva. Verificado con la regresión reintroducida a propósito.
Definición de "hecho" — cumplida
Cambias algo del RAG, corres el harness y obtienes un número que dice si mejoró o empeoró. La evaluación a ojo no cuenta — y aquí no cuenta.
Herramienta interna, no un servicio. El harness no se despliega: se integra. Su valor es proteger a los otros artefactos — de hecho, ya guió una corrección del RAG del Artefacto 2 que sí está en producción. Por eso el contador de "desplegados" sigue en 2/6, pero esta pieza está 100% completa.

Por qué este artefacto importa

Prompt engineering sin evaluación es superstición. Un harness convierte "creo que mejoró" en "mejoró un 14%, sin regresiones" — y es lo que más te distingue en el mercado: no construir el RAG, sino saber demostrar que funciona.
