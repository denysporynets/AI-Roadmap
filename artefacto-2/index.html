RAG sobre el Corpus DOBE
una memoria episódica consultable

Un sistema que responde preguntas sobre un corpus personal citando el fragmento real y su página — y que dice "no guardo ese recuerdo" en vez de inventar cuando la pregunta cae fuera. Desplegado en producción con FastAPI, Docker y Cloud Run. Segundo artefacto de la ruta Data Science → AI Engineering.

Qué hace
Recuperar el momento, no resumir un libro

No es un buscador ni una enciclopedia: es recuperación episódica. Ante una pregunta, encuentra los momentos reales del corpus que más se le parecen por significado, y solo entonces redacta una respuesta anclada a esos momentos, citando de qué pasaje y página sale cada cosa.

La decisión de diseño
El umbral manda antes que el LLM

Antes de llamar al modelo, el sistema mide cuánto se parece la mejor coincidencia. Si no alcanza, no llama al LLM: ni inventa una cercanía que no existe, ni gasta una petición. La honestidad es una característica del diseño, no un accidente.

Cuando sí hay recuerdo (🟢), la respuesta llega redactada y con [#momento · página] en cada afirmación — para que siempre puedas ver de qué pasaje real sale.

De script a servicio
Tres capas y una caja en la nube

El sistema está desacoplado en capas que solo se hablan por un contrato (un JSON). Cambiar la cara no toca el motor; cambiar el modelo no toca la web.

motor
rag_core.py — retrieval + umbral + síntesis. Carga el índice una sola vez. Devuelve un dict, no imprime.
API
FastAPI + uvicorn — /preguntar protegido por token, /salud para el orquestador. Motor cargado en el lifespan.
cara
frontend estático — chat que hace fetch y pinta el JSON: banda 🟢🟡⚪, respuesta y fuentes.
caja
Docker → Cloud Run — imagen mínima construida en la nube (Cloud Build), serverless con escala a cero, claves en Secret Manager.
Privacidad por diseño
Un corpus personal, protegido por capas

El corpus es material personal, así que la privacidad no se deja al azar: el repositorio público no contiene ni el corpus ni los vectores (un .gitignore los blinda), la imagen vive en un registro privado, las claves se inyectan desde Secret Manager (nunca en el código) y la consulta exige token.

Demo real, acceso por token. La URL está viva y abierta para cargar la web, pero preguntar al corpus requiere un token que se comparte de forma selectiva. Así se puede enseñar el sistema funcionando sin abrir el contenido personal a cualquiera.

Por qué este artefacto importa

Un RAG fácil contesta siempre; uno bueno sabe cuándo callar. Poner el umbral antes del modelo convierte "no alucinar" en una decisión de ingeniería — y eso, además, no cuesta dinero.
